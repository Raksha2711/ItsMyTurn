# Backend Instructions

## Technology

- Node.js
- Express
- MySQL

## Backend Standards

- Follow existing project architecture.
- Follow existing naming conventions.
- Reuse services and repositories whenever possible.
- Avoid duplicate logic.

## API Standards

- Use REST APIs.
- Validate all inputs.
- Return consistent response structures.
- Handle exceptions properly.

## Database Rules

- Create migration scripts when needed.
- Follow existing table naming conventions.
- Add indexes where required.
- Maintain referential integrity.

## Security

- Validate request payloads.
- Sanitize inputs.
- Protect sensitive data.
- Follow authentication and authorization patterns already used in the project.

## Before Creating New APIs

Check whether similar APIs already exist.

## Output Expectations

When implementing a module:

- Database changes
- Models
- Repositories
- Services
- Controllers
- Routes
- Validation
- Error handling
- API documentation
